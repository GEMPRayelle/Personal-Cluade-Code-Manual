# Personal-Cluade-Code-Manual
개인용 클로드 코드 설치부터 세팅 가이드 정리

>클로드 설치
1. VSCode나 Cursor등 터미널을 제공하는 환경에서 기본 cmd에서 Powershell로 변경
2. irm https://claude.ai/install.ps1 | iex 명령어 실행
3. claude --version 명령어로 설치가 완료됐는지 확인
4. 설치가 성공적으로 안됐을시 환경 변수 -> 사용자 변수 -> Path 편집 -> ("C:\Users\[Local PC 이름]\.local\bin") 추가
5. 클로드 로그인 후 연동 완료

>플러그인
1. `npm install -g @ryoppippi/ccusage` : CLI에서 일별/월별/실시간 토큰 사용량 확인 가능한 플러그인

>유용한 명령어 + 팁

`/init` : CLAUDE.md 파일을 생성하며 프로젝트 구조를 정리한다

`/btw` : 메인 작업이 멈추지 않고, 백그라운드에서 돌아간다. 이 질문이 히스토리에 들어가지 않음. Space/Enter로 원래 대화로 복귀 가능

-> 메인 대화와 관련없이 토큰을 절약하며 현재 세션 내에서 할 수 있는 질문을 할 수 있다.

`/clear` : 이전 작업과 관련없는 새로운 작업을 시작할때 현재 대화 내역을 초기화해 성능 최적화 + Claude 집중력 향상 (새 작업 시작 전엔 반드시 실행)

`/rename` : /clear를 통해 컨텍스트를 비우기전 세션 이름을 지정
`/resume` : 이전 대화로 쉽게 돌아갈 수 있음 (작업 히스토리가 필요하거나, 나중에 참고해야 할 대화에 유용)

`/context` : 토큰을 어디서 얼마나 쓰는지 컴포넌트별로 확인 (대화 기록, MCP, 파일 로드)

`/cost` : 현재 세션의 실제 토큰 사용량 + 예상 비용

`/usage` : 주간 사용량 한눈에 확인

`/compact` : /context 명령어로 사용률을 확인하고 60% 도달시 보존할 내용을 함께 지시하며 /compact 실행함, Context가 95%차면 자동 압축(Compaction)을 함.
하지만 이때는 이미 컨텍스트 품질이 상당히 저하된 상태임. 압축을 3~4번 해야 할 정도면 세션을 요약 후 /clear 후 새로 시작
> /compact 실행해줘. 현재 구현 중인 기능의 핵심 로직과 발생한 에러 내역은 반드시 보존해줘

`Caching` : 클로드의 캐시 만료 시간은 5분이라 자리를 비울 시 /compact 나 /clear 를 수행하고 자리를 비운다. (5분 후 돌아오면 처음부터 재처리 -> 토큰 사용량 상승)

`#` : 이전에 수행했던 작업을 CLAUDE.md 파일에 추가

`Extended Thinking` : /config 설정 또는 환경변수로 thinking 예산 낮추기가 가능, 복잡한 아키텍처 설계 or 빡센 디버깅일시 켜두는게 좋음

`Shift+Tab` : 자동 수락 모드로 변경

`프롬프트는 범위를 제한` : 파일명, 함수명, 조건을 명시적으로 주는 것이 핵심 (클로드가 알아서 탐색하게 두지 말것)

`간단한 명령은 묶어서 한 번에 보내기` : 요약 + 포인트 뽑고 + 제목 추천 을 한번의 명령으로 수행

`1세션 = 1작업` : 버그3개 고치며 새로운 기능을 추가하고 리팩토링까지 한 세션에 하면 Context가 오염 + 토큰 낭비
